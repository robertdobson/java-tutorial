Basics of File I/O and Stream Objects

Input/Ouput to and from files

  • File input and file output is an essential in programming.
    • Most software involves more than keyboard input and screen user interfaces.
    • Data needs to be stored somewhere when a program is not running, and that means writing data to disk.
    • For this, we need file input and file output techniques.

    • Kinds of Files

    • Formatted Text vs. Binary files
      • A text file is simply made of readable text characters. It looks like the output that is typically printed to the screen through the println statement.
      • A binary file contains unformatted data, saved in its raw memory format. (For example, the integer 123456789 is saved as a 4-byte chunk of data, the same as it's stored in memory - NOT as the 9 digits in this sentence).
    • Sequential vs. Random Access files
      • A sequential file is one that is typically written or read from start to finish
      • A random access file is one that stores records, all of the same size, and can read or write single records in place, without affecting the rest of the file
    • (For now, we'll deal with sequential text files)

An example of reading and writing a text file is demonstrated in the File Input/Output Demo window using the following code from the fileIODemoMethod. The user doesn't need to enter a value. Simply click the 'Submit' button to run the code. After the code completes, the user should be able to find a new file named 'output.txt' containing the same contents read from 'names.txt' in the same directory/folder as the Java Tutorial application.


public static void fileIODemoMethod() {

        try {
            BufferedReader readBuffer = new BufferedReader(new FileReader(inputFile));
            BufferedWriter writeBuffer = new BufferedWriter(new FileWriter(outputFile));

            temp = "\nList of names written to the file \'output.txt\':\n\n";

            while((textSegment = readBuffer.readLine()) != null) {
                temp = temp + textSegment + "\n";
                writeBuffer.write(textSegment);
                writeBuffer.newLine();
            }

            CodeDemoGUI.showResult.setText(temp);

            readBuffer.close();
            writeBuffer.close();
        }
        catch (FileNotFoundException unableToOpen) {
            CodeDemoGUI.showResult.setText("Unable to open file '" + inputFile + "'");
        }
        catch (IOException errorReading) {
            CodeDemoGUI.showResult.setText("Error reading file '" + inputFile + "'");
        }

    }

What's the code doing?

The fileIODemoMethod is called from the CodeDemoGUI. The code then tries to read from the input file 'names.txt' while it is not null (while something is available to be read), builds a string to output the file contents read and written to the text area in the File Input/Output Demo window, writes the file contents to a new file called 'output.txt', and closes the readBuffer and writeBuffer. If it is unsuccessful (unable to open or encounters an error reading the file), then an exception error message is sent to the text area of the File Input/Output Demo window.
