What is a loop?

A loop is a sequence of statements which is specified once but which may be carried out several times in succession. The code "inside" the loop (the body of the loop) is obeyed a specified number of times, or once for each of a collection of items, or until some condition is met, or indefinitely.

Which LOOP should I use????

  • while:  the loop must repeat until a certain "condition" is met.  If the "condition" is FALSE at the beginning of the loop, the loop is never executed.  The "condition" may be determined by the user at the keyboard.  The "condition" may be a numeric or an alphanumeric entry.  This is a good, solid looping process with applications to numerous situations.

 

  • do-while:  operates under the same concept as the while loop except that the do-while will always execute the body of the loop at least one time.  (Do-while is an exit-condition loop -- the condition is checked at the end of the loop.)  This looping process is a good choice when you are asking a question, whose answer will determine if the loop is repeated.

 

  • for:  the loop is repeated a "specific" number of times, determined by the program or the user.  The loop "counts" the number of times the body will be executed.  This loop is a good choice when the number of repetitions is known, or can be supplied by the user.

A loop is demonstrated with while statements in the Loop Demo window using the following code from the loopDemoMethod. The user selects the amount of change they should receive between 1 and 99 cents, and clicks the 'Submit' button.


public static void loopDemoMethod(int userEntry) {

        while(userEntry > 49) {
            userEntry -= 50;
            ++halfDollars;
        }

        while(userEntry > 24) {
            userEntry -= 25;
            ++quarters;
        }

        while(userEntry > 9) {
            userEntry -= 10;
            ++dimes;
        }

        while(userEntry > 4) {
            userEntry -= 5;
            ++nickels;
        }
        CodeDemoGUI.showResult.setText("Your change is " +
                halfDollars + " half dollar(s), " + quarters +
                " quarter(s), " + dimes + " dime(s), " + nickels +
                " nickel(s), and " + userEntry + " penn(y/ies)!");

        // Reset global variables
        halfDollars = 0;
        quarters = 0;
        dimes = 0;
        nickels = 0;
    }

What's the code doing?

The loopDemoMethod is called from the CodeDemoGUI and passed an int argument named userEntry, which is a value taken from the user via the spinner. The code then processes the information through a series of while loops, incrementing the appropriate coins, and decrementing userEntry by the value of the corresponding coin. Then a message is output to the text area in the Loop Demo window declaring the correct change and coin count.
